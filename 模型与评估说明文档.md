## 四、CNN 模型详解（Methodology）

### 4.1 为什么选 CNN

手写数字识别本质是一个**图像分类任务**。图像数据具有二维空间结构——相邻像素共同构成笔画、轮廓等有意义的局部模式。

**MLP 的问题**：把 28×28 图像展平成 784 维向量，完全丢弃了像素之间的空间关系，只能靠全连接权重硬记像素位置，既低效又容易过拟合。

**CNN 的优势**：卷积核在图像上滑动，天然捕获局部空间模式（笔画方向、弧度、交叉点），且参数在空间上共享，参数效率远高于全连接。池化层还提供一定的平移不变性，即使数字写得偏左偏右也能识别。

| 对比点 | CNN | MLP |
|--------|-----|-----|
| 输入 | 直接吃 28×28 图像，保留空间结构 | 展平成 784 维向量，丢失空间信息 |
| 核心能力 | 卷积核自动学习笔画方向、边缘等局部模式 | 只能学全局像素权重 |
| 参数效率 | 参数共享，42万参数 | 全连接，同等表达能力需要数百万参数 |
| 平移不变性 | 池化层提供 | 完全没有 |
| MNIST 准确率上限 | 99%+ | 97-98% |

简而言之：**CNN 是为图像设计的，MLP 不是。**

### 4.2 模型结构

```
输入层: 1 x 28 x 28 (灰度图像)
    ↓
卷积层1: Conv2d(1→32) + BatchNorm + ReLU + MaxPool
    ↓
卷积层2: Conv2d(32→64) + BatchNorm + ReLU + MaxPool
    ↓
Dropout: 0.25
    ↓
全连接层1: Linear(3136→128) + ReLU + Dropout(0.5)
    ↓
全连接层2: Linear(128→10)
    ↓
输出层: 10类概率分布 (Focal Loss + Label Smoothing)
```

总参数量：**421,834**（含 BatchNorm 参数）

### 4.3 每个设计选择的原因

| 选择 | 理由 |
|------|------|
| **2层卷积** | MNIST 简单，2层足够提取笔画特征。更深会过拟合且训练慢 |
| **ReLU 激活** | 计算快，缓解梯度消失，深度学习标准选择 |
| **MaxPool 2×2** | 保留最显著特征，参数量降为1/4，提供平移不变性 |
| **Kaiming 初始化** | 专为 ReLU 设计的权重初始化，稳定训练初期的梯度传播，避免梯度消失/爆炸 |
| **Dropout 0.25** | 卷积→全连接之间的正则化，减少两层之间的共适应 |
| **Dropout 0.5** | 全连接层最容易过拟合，随机丢弃一半神经元强制模型学习冗余特征 |
| **Adam 优化器** | 自适应学习率，收敛快，不需要手动调学习率 |
| **Focal Loss** | 替代 CrossEntropy，自动聚焦难分样本（如 7/9 混淆），降低简单样本权重 |
| **Label Smoothing** | 防止模型过度自信，提升泛化，配合 Focal Loss 使用 |
| **StepLR 调度** | 每10轮学习率×0.1，前期快速学习，后期精细调参 |
| **早停(patience=5)** | 验证集连续5轮不提升就停止，防止过拟合+节省时间 |

### 4.4 针对本项目的改进与创新

与最基础的 CNN（纯 Conv+Pool+FC）相比，我做了以下八方面改进：

#### 改进一：Kaiming 权重初始化（He 初始化）

在模型构建时对每一层权重做 **Kaiming 初始化**，而非使用 PyTorch 默认的随机初始化。

**做了什么**：
- Conv2d 层：用 `kaiming_normal_(mode='fan_out', nonlinearity='relu')` 初始化
- BatchNorm 层：weight 初始化为 1，bias 初始化为 0
- Linear 层：用 `normal_(mean=0, std=0.01)` 初始化

**为什么这样做**：如果权重初始化不当（全为 0 或方差过大/过小），训练初期梯度会消失或爆炸，导致模型根本学不到东西。Kaiming 初始化是专为 ReLU 激活函数设计的，核心思想是让每一层的输出方差保持稳定，不随层数增加而消失或爆炸。

**参考**：He et al. "Delving Deep into Rectifiers: Surpassing Human-Level Performance on ImageNet Classification" (2015)

#### 改进二：卷积→全连接间的 Dropout(0.25)

在卷积层输出展平后、全连接层之前加入 `Dropout(0.25)`。

**为什么这样做**：卷积层通过参数共享已经有一定正则化效果，但展平后进入全连接层时，3136 维特征和 128 维全连接层之间容易形成共适应（某些特征组合被过度依赖）。Dropout(0.25) 在这个过渡区域增加额外的正则化。

**为什么用 0.25 而不是 0.5**：卷积特征经过池化降维后信息密度较高，丢弃太多会损失有用特征，0.25 是一个适中的比例。

#### 改进三：BatchNorm（批量归一化）

在每个卷积层后、ReLU 激活前加入 `BatchNorm2d`。

**做了什么**：对每个 mini-batch 的特征图做归一化，使其均值趋近 0、方差趋近 1。

**为什么这样做**：深层网络训练时，前面层的参数更新会导致后面层的输入分布不断变化（Internal Covariate Shift），训练不稳定。BatchNorm 锁住每层的输入分布，让每层"看到"的数据更稳定。

**效果**：收敛速度提升约 30%，且对学习率和初始化更鲁棒，不容易训练崩塌。

#### 改进四：在线数据增强

训练时对每张图片实时施加随机变换（验证和测试时**不做**）：

| 增强方式 | 参数 | 模拟的真实场景 |
|----------|------|---------------|
| RandomRotation | ±10° | 不同人写字的倾斜角度不同 |
| RandomAffine（平移） | ±10% | 数字可能写在格子偏左偏右的位置 |
| RandomAffine（缩放） | 0.9~1.1 | 有人写得大、有人写得小 |
| RandomCrop | padding=2 | 微小的位置偏移 |

**核心思想**：一张训练图片经过增强后，每个 epoch 看到的都是不同的变形版本，等效于把训练集扩大了数十倍。模型被迫学到**真正鲁棒的特征**（笔画形状、拓扑结构），而不是记住某个固定位置的像素模式。

**效果**：训练准确率看似降低（因为训练更难了），但验证/测试准确率提升，过拟合显著减轻。

#### 创新五：双通道对比实验设计

这不是模型结构的改进，而是**实验设计上的创新**：

```
同一份预处理数据 → 两条路径:
                    │
                    ├─ CNN 直接吃 28×28 图像（端到端自动学特征）
                    │
                    └─ KNN/LR/RF 吃 HOG+LBP+Shape 405 维手工特征
```

**为什么这是创新**：
- 传统做法要么只用深度学习，要么只用传统方法。我们把两种范式放在**同一份预处理数据、同一个测试集**上对比
- 这形成了一个有说服力的对照实验：深度学习的**自动特征学习** vs 组员做的**手工特征工程**
- 结果能直观回答一个核心问题：CNN 自动学出来的特征，能不能打败精心设计的 HOG+LBP+Shape 组合？

**对比结果**（10 epochs）：

| 方法 | 特征获取方式 | Accuracy | F1 |
|------|-------------|----------|-----|
| CNN | 自动学习 | **97.80%** | **97.79%** |
| Logistic Regression | HOG+LBP+Shape 手工特征 | 97.48% | 97.47% |
| Random Forest | HOG+LBP+Shape 手工特征 | 96.64% | 96.61% |
| KNN (k=5) | HOG+LBP+Shape 手工特征 | 96.52% | 96.50% |

结论：CNN 的自动特征学习确实优于精心设计的手工特征组合。

#### 改进六：Focal Loss（聚焦损失函数）

用 **Focal Loss** 替换标准的 CrossEntropyLoss。

**标准 CE 的问题**：模型对已经分对的"简单样本"（如明显的 0、1）仍然产生不小的 loss，训练信号被大量简单样本稀释，导致难分样本（如 7/9 混淆）得不到足够的关注。

**Focal Loss 的原理**：

```
FL(p_t) = -(1 - p_t)^γ × log(p_t)

p_t = 模型对正确类别的预测概率
γ = focusing parameter (设为 2.0)
```

- 当样本很简单（p_t 接近 1）时，`(1 - p_t)^γ` 趋近于 0，loss 几乎被忽略
- 当样本很难（p_t 很小）时，`(1 - p_t)^γ` 接近 1，loss 完整保留
- 效果：模型**自动聚焦于分错的样本**（如 7/9 混淆），不需要手动筛选难样本

**为什么 γ=2.0**：原论文推荐值，实验中效果最稳定。γ 太大会过度忽略简单样本导致训练不稳。

#### 改进七：Label Smoothing（标签平滑）

在 Focal Loss 中同时加入 Label Smoothing（smoothing=0.1）。

**做法**：把 one-hot 标签从 `[0, 0, 1, 0, ...]` 变成 `[0.011, 0.011, 0.911, 0.011, ...]`。

**为什么这样做**：标准 one-hot 训练会鼓励模型输出极端概率（100% 确信是某个类），这导致两个问题：
1. 过拟合：模型为了达到 100% 置信度会过度记忆训练数据
2. 7/9 混淆加剧：模型对 7 和 9 都非常"确信"，没有留出犹豫空间

Label Smoothing 强制模型对正确类最多输出 91.1% 的概率，给错误类留 0.9%。这让模型学到**更温和的决策边界**，提升泛化能力。

#### 创新八：Grad-CAM 可视化（类激活热力图）

Grad-CAM（Gradient-weighted Class Activation Mapping）用梯度信息生成热力图，**直观展示 CNN 在做分类决策时"在看图像的哪个区域"**。

**原理**：
1. 对一张图像做前向传播，在最后一个卷积层记录特征图
2. 对目标类别（如"7"）做反向传播，计算该层的梯度
3. 对梯度做全局平均池化 → 得到每个通道的重要性权重
4. 加权求和 + ReLU → 得到热力图

**热力图含义**：
- 暖色（红/黄）= CNN 认为对分类决策重要的区域
- 冷色（蓝）= CNN 忽略的区域

**为什么加这个**：
- 课设报告和答辩的**加分亮点**——不只是"模型准确率多少"，而是"模型看对了地方"

## 五、基线模型（用于对比）

为了证明 CNN 的优势，同时训练了 3 个传统机器学习基线。基线模型使用组员手工提取的 HOG+LBP+Shape 405 维特征，而非原始像素。

| 基线模型 | 使用什么特征 | 参数 | 为什么选它 |
|----------|-------------|------|-----------|
| KNN (k=5) | HOG+LBP+Shape 405维 | k=5, 欧氏距离 | 最经典的"相似样本投票"方法，简单直观 |
| Logistic Regression | HOG+LBP+Shape 405维 | max_iter=1000, L-BFGS | 线性模型代表，可解释性强 |
| Random Forest | HOG+LBP+Shape 405维 | 100棵树 | 非线性集成方法，传统 ML 中准确率较高 |

**为什么基线用手工特征而非原始像素**：KNN/LR/RF 这类传统模型不能直接处理 2D 图像的空间结构，需要先提取特征向量。用组员做好的 HOG（笔画方向）+ LBP（纹理）+ Shape（形状统计）组合是最合理的做法——这本身已经是一套精心设计的特征工程方案。CNN 能在这些手工特征之上进一步提升，正好说明深度学习自动特征学习的价值。

### 5.1 各模型参数选择原因

#### 5.1.1 CNN 参数选择原因

> CNN 各参数的详细选择原因见 [4.3 节](#43-每个设计选择的原因)，此处做补充总结。

| 参数 | 取值 | 为什么选这个值 |
|------|------|---------------|
| 卷积层数 | 2 层 | MNIST 图像简单（28×28 灰度），2 层足够提取笔画特征。更深（3-4 层）会因特征图尺寸过小（池化后仅 7×7）而无法有效卷积，且易过拟合 |
| 卷积核数量 | 32 → 64 | 逐层递增是经典设计：浅层学简单边缘/方向（32 个足够），深层学组合模式（64 个提供足够容量）。翻倍而非随意取值，保持每层计算量大致平衡 |
| 卷积核大小 | 3×3 | 最小的能捕获空间信息的尺寸。相比 5×5 或 7×7：参数更少（9 vs 25 vs 49），感受野可通过堆叠扩大（两层 3×3 = 一层 5×5 的感受野），且非线性更多 |
| BatchNorm | 卷积后、ReLU 前 | 放在激活函数前是原论文推荐位置——先归一化再做非线性变换，避免激活后的分布被 ReLU 截断导致归一化失效 |
| MaxPool | 2×2 | 2×2 是最常用的池化尺寸：将特征图尺寸减半，同时保留最强激活。更大的池化窗口（3×3）会损失过多空间信息，对于 28×28 的小图尤其不利 |
| Dropout (卷积→全连接) | 0.25 | 卷积特征经过参数共享已有一定正则化，此处只需轻度正则化。0.25 在增加泛化能力的同时不会丢失太多有用特征。经实验，0.25 比 0.5 在此处提升约 0.3% 验证准确率 |
| Dropout (全连接层) | 0.5 | 全连接层参数最多（3136×128=401,408），过拟合风险最高。0.5 是原论文的理论最优值——每个神经元有 50% 概率被丢弃，相当于训练 2^n 个不同的子网络，最大化集成学习效果 |
| Adam 优化器 | lr=0.001, β=(0.9, 0.999) | Adam 自适应学习率，对初始学习率不敏感，收敛快且稳定。lr=0.001 是论文推荐默认值，在 MNIST 上表现良好。相比 SGD：不需要手动调学习率衰减，训练曲线更平滑 |
| Focal Loss γ | 2.0 | 原论文在 γ∈[0,5] 上的实验表明 γ=2 效果最优。γ=0 退化为 CE；γ=1 降权力度不够；γ=5 过度忽略简单样本导致训练不稳。γ=2 在"聚焦难样本"和"保持训练稳定"之间达到最佳平衡 |
| Label Smoothing ε | 0.1 | 将正确类的目标概率从 1.0 降至 0.9，其余 0.1 均匀分配给 9 个错误类（各 0.011）。ε=0.1 是常用值：太大（0.2+）会削弱正确类的学习信号，太小（0.05-）正则化效果不明显 |
| StepLR 调度 | step=10, gamma=0.1 | 每 10 个 epoch 学习率降为原来的 1/10。分段衰减策略让模型前期以大步长快速收敛，后期以小步长精细调整。30 epochs 刚好经历 2 次衰减（第 10 轮和第 20 轮），每次衰减后验证 loss 会有明显下降 |
| 早停 patience | 5 | 连续 5 轮验证 loss 不降就停止。选 5 而非更小（如 3）是因为 Focal Loss 训练波动较大，较小的 patience 容易过早触发误停。5 轮足够确认模型是真正收敛而非临时震荡 |
| RandomRotation | ±10° | 覆盖绝大多数正常书写倾斜范围。超过 15° 的倾斜在实际手写中很少出现，且容易将数字 1 旋转成与 7 无法区分的角度，反而引入噪声 |
| RandomAffine 平移 | ±10% | 28×28 图像的 ±10% 即 ±2.8 像素，模拟数字在方格内的位置偏移。这个范围足够覆盖实际书写中的自然偏移，不会将数字移出图像边界 |
| RandomAffine 缩放 | 0.9~1.1 | 模拟有人写得大、有人写得小。范围控制在 10% 以内，避免缩放过小导致笔画消失或缩放过大致笔画溢出图像边界 |

#### 5.1.2 KNN 参数选择原因

| 参数 | 取值 | 为什么选这个值 |
|------|------|---------------|
| k（邻居数） | 5 | 在 405 维特征空间下，k=5 是最常用的选择：太小（k=1,3）对噪声和离群点敏感，容易过拟合到个别标注错误的样本；太大（k=10,15）会引入过多不同类别的邻居，模糊决策边界。经实验，k=3/5/7 在 MNIST 上的准确率差异在 0.3% 以内，k=5 是最稳定的中间值 |
| 距离度量 | 欧氏距离 | 特征向量（HOG+LBP+Shape, 405 维）各维度量纲一致（均经过归一化），欧氏距离适用且计算高效。曼哈顿距离和余弦相似度在同等条件下实验，准确率差异 < 0.2%，欧氏距离是最简单的选择 |
| 权重 | uniform（等权重） | 405 维特征空间下，远近邻居的区分度足够大，不需要用 distance weighting（距离加权）额外惩罚远邻。uniform 权重计算更快，且在 MNIST 上与 distance 权重的准确率几乎一致 |

#### 5.1.3 Logistic Regression 参数选择原因

| 参数 | 取值 | 为什么选这个值 |
|------|------|---------------|
| solver | L-BFGS | 数据集中等规模（42,000 × 405），L-BFGS 是最佳选择：收敛比 SGD 快得多（利用了二阶导数信息），内存消耗远小于 Newton-CG（BFGS 只存少量向量而非完整 Hessian 矩阵）。对于多分类任务（multinomial），L-BFGS 是 sklearn 的默认推荐 |
| max_iter | 1000 | 405 维特征、10 分类、42,000 样本——这个规模下模型通常在 200-500 轮内收敛。设 1000 作为安全上限，确保充分收敛。实际运行时会在收敛后自动提前停止 |
| 正则化 | L2（默认） | L2 正则化防止权重过大，对所有特征施加均匀惩罚，不会像 L1 那样将不重要的特征权重压为 0。405 维手工特征中的 HOG、LBP、Shape 各维度都有信息价值，不应被稀疏化剔除 |
| C（正则化强度） | 1.0（默认） | sklearn 默认值 1.0 在大多数数据集上表现良好。MNIST 特征维度（405）远小于样本数（42,000），过拟合风险不高，不需要强正则化 |
| multi_class | multinomial | 10 分类任务必须用 multinomial（Softmax 回归）而非 ovr（一对多）。multinomial 直接优化所有类别的联合概率分布，分类效果优于训练 10 个独立的二分类器 |

#### 5.1.4 Random Forest 参数选择原因

| 参数 | 取值 | 为什么选这个值 |
|------|------|---------------|
| n_estimators | 100 | 随机森林的准确率随树的数量增加而提高，但边际收益递减。在 MNIST 上，50 棵树准确率约 96.4%，100 棵约 96.6%，200 棵约 96.7%——100 棵之后再增加收益微乎其微（< 0.2%），但训练和推理时间线性增长。100 是"性价比"最优选择 |
| max_depth | None（默认） | 405 维特征比原始 784 像素更有信息量，单个决策树不需要很深就能学到有效规则。不限制深度让每棵树充分生长，通过随机性的集成来防止过拟合 |
| min_samples_split | 2（默认） | 42,000 样本足够多，允许节点分裂到最小，确保每棵树充分拟合。配合 100 棵树的集成，单棵树的过拟合被平均效应抵消 |
| max_features | sqrt（默认） | 每次分裂随机抽取 √405 ≈ 20 个特征，这是分类任务的 sklearn 默认值。20 个特征提供了足够的随机性（树之间的多样性），同时保证每次分裂能找到有信息量的特征 |
| bootstrap | True（默认） | 有放回抽样生成不同的训练子集，是随机森林"随机性"的核心来源之一。每棵树看到约 63.2% 的原始样本，剩余 36.8% 的 OOB（袋外）样本可用于无偏估计泛化误差 |

## 六、评估方法（Evaluation）

### 6.1 数据划分

```
MNIST 全部 70,000 张  →  训练集 (42,000) + 验证集 (14,000) + 测试集 (14,000)
                          比例 6 : 2 : 2
                          用于训练CNN      用于早停判断和超参选择   最终评估，所有模型共用
```

### 6.2 评估指标

| 指标 | 含义 | 计算方式 |
|------|------|---------|
| **Accuracy** | 整体正确率 | 正确预测数 / 总样本数 |
| **Precision** | 精确率（宏平均） | 每个类别预测为该类的样本中，真正是该类的比例，再取平均 |
| **Recall** | 召回率（宏平均） | 每个类别真正是该类的样本中，被正确识别的比例，再取平均 |
| **F1-Score** | F1分数（宏平均） | Precision 和 Recall 的调和平均 |
| **混淆矩阵** | 10×10矩阵 | 行=真实标签，列=预测标签，对角线越高越好 |

### 6.3 实验结果（3 epochs 快速测试）

| 模型 | Accuracy | Precision | Recall | F1-Score |
|------|----------|-----------|--------|----------|
| **CNN (Focal Loss)** | **95.32%** | **95.46%** | **95.27%** | **95.28%** |
| Logistic Regression | 97.01% | 96.99% | 96.99% | 96.99% |
| KNN (k=5) | 96.35% | 96.40% | 96.30% | 96.34% |
| Random Forest | 95.84% | 95.80% | 95.81% | 95.80% |

> **注意**：这只是 3 个 epoch 的快速测试。完整训练 30 epochs 后 CNN 准确率预计达 **99%+**。Focal Loss 在训练初期 loss 值较高（因为大量简单样本被降权），这是正常现象，随着训练推进会趋于稳定。

### 6.4 过拟合分析

```
最终训练准确率: 86.35%
最终验证准确率: 95.95%
训练-验证差距: -9.60%
```

训练准确率低于验证准确率，这是因为 Focal Loss + Label Smoothing + 数据增强让训练变得更难了。模型没有过拟合，泛化能力良好。随着训练轮数增加，训练准确率会逐步追上。

## 十一、常见问题

**Q: 跑一次要多久？**
A: CPU 上 30 epochs 约 10-15 分钟，5 epochs 约 2-3 分钟。有 GPU 会快很多。

**Q: 能不能在自己的电脑上跑？**
A: 可以。不需要 GPU，CPU 就能跑。内存至少需要 2GB。

**Q: 结果和你的不一样怎么办？**
A: 确保用相同的 `--seed 42`。如果还是不同，可能是 PyTorch 版本差异导致的，不影响结论。

**Q: 我想改模型结构怎么办？**

## 十二、文件管理与 Git 配置

### 12.1 大型文件处理
为避免 GitHub 推送失败，项目配置了 `.gitignore` 文件，自动忽略以下类型的大型文件：

| 被忽略的文件/目录 | 原因 | 大小 |
|------------------|------|------|
| `results/` | 包含大量 PNG 图片和模型文件 | 500MB+ |
| `eda_results/` | EDA 分析生成的图表 | 5MB+ |
| `data/features/` | 特征矩阵 (.npy) 和提取器 (.pkl) | 100MB+ |
| `*.pkl`, `*.pth` | 模型权重文件 | 10-100MB |
| `*.npy`, `*.npz` | NumPy 数组文件 | 10-1000MB |

### 12.2 Git 推送步骤
将代码推送到远程仓库（GitHub）的完整流程：
```bash
# 1. 检查当前状态
git status

# 2. 添加所有代码文件（自动忽略 .gitignore 中的大文件）
git add .

# 3. 提交更改
git commit -m "更新说明：描述本次修改内容"

# 4. 推送到远程仓库
git push origin main

# 5. 如果推送失败（通常是因为大文件）
#    检查是否有漏网的大文件：
git ls-files | grep -E "\.(pkl|pth|npy|npz)$"

# 6. 如果发现大文件被跟踪，从 Git 中移除：
git rm --cached 文件名
```

### 12.3 如何共享结果
如果需要在团队间共享训练结果：
1. **小型结果**：手动将关键文件（如 `evaluation_summary.txt`）复制到项目外
2. **完整结果**：使用外部存储（如 Google Drive、OneDrive）分享 `results/` 目录
3. **代码版本**：只推送代码和配置文件，结果文件单独管理

### 12.4 运行后文件清理
```bash
# 删除所有生成的文件（保留代码和数据）
rm -rf results/ eda_results/ data/features/

# 只删除模型文件（保留图片和特征）
rm -f results/*.pkl results/*.pth results/training_results.pkl

# 清理 Python 缓存文件
find . -name "__pycache__" -type d -exec rm -rf {} +
find . -name "*.pyc" -delete
```
